§ 04 — ARCHITECTURE TECHNIQUE
Système Intégral

Système Intégral
en 5 Phases

Le système fonctionne en 5 phases interconnectées, de la capture de données client jusqu'à l'action commerciale validée. Chaque phase est auditée et traçable.

PHASE 01 Expérience Client Intelligente — Interface Adaptative
▼

Le système ne présente pas la même chose à tout le monde. Il observe, s'adapte, et personnalise en temps réel.

MODULE 1 : Simulateur financier intelligent (ResultsDashboard)

  • Inflation 5% annuelle, prix kWh évolutif, taille habitation, orientation toiture
  • ROI précis, cashflow mensuel, point mort sur 3 scénarios (10, 15, 20 ans)
  • Validation solvabilité : taux d'endettement max 35%, revenus déclarés
  • Comparaison avec/sans installation : économies année par année

MODULE 2 : Profilage psychologique comportemental

  • Hésitation : temps de réponse sur chaque question
  • Besoin de preuves : clics sur 'Voir détails', 'Certifications'
  • Peur de l'erreur : retours en arrière, re-lecture
  • Sensibilité au prix : temps passé sur tarif

MODULE 3 : Agent Zero (Cerveau d'adaptation temps réel)

  • Réorganise l'ordre des modules selon profil détecté
  • Règle vitesse d'affichage (750ms rassure, 200ms dynamise)
  • Adapte vocabulaire (technique / simplifié / urgent)
  • Décide du niveau de détail (masque ou expose)
PHASE 02 Stockage & Tracking — Supabase PostgreSQL
▼

Base de données centrale stockant vérité commerciale et comportementale.

  • Table 'studies' : simulations financières avec statut (simulated → signed → paid)
  • Table 'tracking_events' : vues, clics, temps passé → détection état 'Agité' ou 'Muet'
  • Table 'email_queue' : file d'attente emails validés par Guards
  • Table 'decision_logs' : boîte noire inviolable de toute recommandation système (audit trail complet)
PHASE 03 Cerveau Décisionnel — SystemBrain TypeScript
▼

Intelligence locale tournant dans dashboard admin. Analyse et recommande.

1. Analyse des signaux (Engine)

  • Danger Score (0–100) par dossier : (Poids Temps) + (Poids Valeur) + (Poids Comportement)
  • États : Muet (0 réaction), Agité (stress sans clic), Intéressé (clics actifs), Fatigué (trop de relances)

2. Segmentation stratégique (3 axes)

  • AXE A (Signés) : priorité absolue, sécurisation acompte
  • AXE B (Pipeline) : opportunités chaudes à closer
  • AXE C (Leads) : vivier à nurturer avec parcimonie
PHASE 04 Agents Autonomes de Gouvernance — Ops Agent
▼

Couche de sécurité empêchant toute action non-conforme. Aucune recommandation n'atteint le commercial sans validation de TOUS les Guards.

  • Temporal Guard : bloque si < 24h depuis dernier contact (prévention harcèlement)
  • Fatigue Guard : bloque si client 'Fatigué' (>3 relances sans réponse)
  • Integrity Guard : vérifie cohérence données (aucune donnée orpheline)
  • Privacy Guard : bloque tout si opt-out RGPD détecté
  • OpsAudit : génère PDF de conformité prouvant respect des règles
PHASE 05 Interface Décisionnelle — Dashboard Admin
▼

4 vues selon rôle utilisateur :

  • Cockpit (Pilote) : UNE seule info → 'Qu'est-ce qui brûle maintenant ?'
  • War Room (Exécution) : tableau tactique AXE A/B, actions [Relancer] [Acompte]
  • Pilotage (Stratège) : graphiques performance, cartes thermiques de risque
  • Registres (Comptable) : accès brut aux tables, export, analyse forensique

Stack Technique

  • Front-end : React + TypeScript + Vite (Vercel, CDN global)
  • Base de données : Supabase PostgreSQL + Edge Functions
  • IA Core : Python Flask (PythonAnywhere) avec clé API sécurisée
  • Connexion : API REST HTTPS-only, authentification JWT
  • Logs : Table decision_logs = boîte noire inviolable (append-only)

OBJ 02"On a déjà un CRM"
▼

Un CRM vous dit ce qui s'est passé (historique). Notre système vous dit ce qui va casser si vous n'agissez pas maintenant (prédictif). C'est complémentaire.

▼
